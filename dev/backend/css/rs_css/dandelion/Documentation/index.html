Introduction

Dandelion Admin is a responsive, full featured admin template build exclusively for sale at ThemeForest.
This template focuses on having a simple, clear design, while providing the most commonly used elements on web apps.

Key Features

  • Pixel perfect responsive design, this templates renders perfectly even when displayed on smart phones, tablets and wide screen monitors
  • Two layout modes: fixed and fluid
  • Scrollable panels that works on touch devices
    The most common problem for mobile devices is to display content that expands horizontally, this template provides a solution by including a feature to add a horizontal scrollbar below the content by explicitly setting a css class.
  • Three custom plugins:
    • Animated circular statistics widget

    • Picklist, a dual select jquery plugin

File & Folder Structure

Before going into details, lets look at how this template's files and folders are structured.

Folder Structure

1. css
   This folder is the root folder where all css stylesheets are located
   1.1 core
       This folder contains all core css files that styles every core elements

2. gallery
   This folder contains all gallery images and thumbnails for the demo page, it's safe to delete this folder

3. images
   This folder contains all images and icons
   3.1 bg
       This folder contains all body background patterns
   3.2 headers
       This folder contains all header background patterns
   3.3 icons
       This folder contains all icon files, black, white, and color with 16x16 and 32x32 sizes.

4. js
   This folder contains all JavaScript files, which are the startup script and some plugin files
   4.1 core
       This folder contains all core JavaScript files, that is required by this template to function properly
       4.1.1 plugins
             This folder contains the three custom plugins build for this template
   4.2 demo
       This folder contains all javascript files needed to run the demo on every page, it's safe to delete this folder if you don't want to use the demo files

5. jui
   This folder contains all css, javascript and images for jquery-ui

6. plugins
   This folder contains all external plugin files which are:
   - jquery chosen plugin for styling select inputs
   - jquery colorpicker plugin
   - jquery datatables plugin, an extensive, rock solid table plugin
   - jquery elastic plugin, for auto growing textareas
   - elfinder for easy file management
   - elrte, a WYSIWYG text editor
   - Fullcalendar, calendar plugin
   - jgrowl, MacOSx notification style plugin
   - photoswipe, a gallery plugin optimized for touch devices
   - placeholder plugin a fallback for HTML5 placeholders on IE
   - plupload, a file uploading plugin
   - prettyphoto, a image displaying plugin
   - syntaxhighligter, a plugin that outputs beautiful syntax highlighting
   - jquery tipsy, a tooltip plugin
   - jquery validate plugin

Core CSS Files

Most of this template elements are styled by the CSS files located in /css/core, there are in total 15 files in this folder, each file styling a specific element.

These CSS files are included indirectly by default on every page by including /css/dandelion.css.

/* The Contents of /css/dandelion.css */

@import url("core/typography.css");
@import url("core/core.css");
@import url("core/panel.css");
@import url("core/form-structure.css");
@import url("core/form-elements.css");
@import url("core/form-message.css");
@import url("core/button.css");
@import url("core/table.css");
@import url("core/plugins.css");
@import url("core/gallery.css");
@import url("core/statistic.css");
@import url("core/panel-toolbar.css");
@import url("core/wizard.css");
@import url("core/error.css");

/* Customizer stylesheet, remove if not needed */
@import url("core/customizer.css");

The CSS files are separated in order to increase readability, and ease maintenance.

In case you want to merge these files into one file to minimize HTTP requests, there's already a file that merges all those CSS files into one file, which is /css/dandelion.min.css.

This file is actually a merged version of all these CSS files, it's recommended to use this file when in production mode :)

Below is a list of all CSS files, along with a short description of what each file contains:

  • button.css
    Contains general button styling, with six color schemes (red, green, blue, orange, gray and pink)
  • core.css
    Contains the main layout styles, (body, header, navigation, content area and footer)
  • customizer.css
    Contains the customizer layout style, this file can be removed if not needed
  • error.css
    Contains the error page styles
  • form-elements.css
    Contains all form element styles (input, textarea, select, custom file input)
  • form-message.css
    Contains all block message styles (error, warning, info and success)
  • form-structure.css
    Contains the main form structure styles
  • gallery.css
    Contains the gallery thumbnail styles and the image link styles
  • panel-toolbar.css
    Contains the panel toolbar css class
  • panel.css
    Contains the content sections css classes, (plain content, panel content, scrollable content)
  • plugins.css
    Contains css style overrides to force plugin styles to match this template
  • statistic.css
    Contains css styles for statistical widgets
  • table.css
    Contains css styles for tables, and jquery DataTables plugin
  • typography.css
    Contains this template's typography styles
  • wizard.css
    Contains css styles for the wizard navigation, which is utilized by this template's custom Wizard plugin

Note: Media queries doesn't have it's own stylesheet, instead they are placed on the bottom of each CSS file.

Building the base HTML

This whole template is coded with XHTML markup and CSS3.

Building the base HTML structure is pretty straightforward. On this section, we'll go through a complete process on creating a complete page that includes the minimal required javascript and css files.

The Default XHTML Header With The Necessary Metatags

<!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.0 Transitional//EN" "http://www.w3.org/TR/xhtml1/DTD/xhtml1-transitional.dtd">
<html xmlns="http://www.w3.org/1999/xhtml">
<head>
<meta http-equiv="Content-Type" content="text/html; charset=utf-8" />

<!-- Viewport metatags -->
<meta name="HandheldFriendly" content="true" />
<meta name="MobileOptimized" content="320" />
<meta name="viewport" content="width=device-width, initial-scale=1.0, maximum-scale=1.0, user-scalable=no" />

<!-- iOS webapp metatags -->
<meta name="apple-mobile-web-app-capable" content="yes" />
<meta name="apple-mobile-web-app-status-bar-style" content="black" />

<!-- iOS webapp icons -->
<link rel="apple-touch-icon" href="touch-icon-iphone.png" />
<link rel="apple-touch-icon" sizes="72x72" href="touch-icon-ipad.png" />
<link rel="apple-touch-icon" sizes="114x114" href="touch-icon-retina.png" />

There are three metatags section on the code above:

  1. Viewport metatags, are all required metatags for this template to display correctly on mobile phones and tablets.
  2. iOS webapp metatags, are all required metatags that makes this template webapp compatible, where the default iOS headers will be hidden on iOS
  3. iOS webapp icons, are all required metatags that makes this template accessible as an icon on the homescreen of iOS devices.

Including The Core CSS, jQuery and Core JavaScript Files

<link rel="stylesheet" type="text/css" href="css/reset.css" media="screen" />
<!--  Fluid Grid System -->
<link rel="stylesheet" type="text/css" href="css/fluid.css" media="screen" />
<!--  Main Stylesheet -->

There are three css stylesheets that can't be missed, two of them is not part of the core styles, but necessary to get this template to display correctly:

  1. CSS reset,  a stylesheet that resets the default browser stylesheet, to make this template looks consistent on all browsers.
  2. Fluid grid system, a stylesheet that contains fluid grid system, for dividing content into grids
  3. Main stylesheet, contains all core css styles

At last, the main JavaScript file that contains all necessary startup code and template functions is included.

Grid and Content

The Grid

The contents on this template are divided into grids. The grids are fluid and responsive, meaning that they will adapt to screen size, and they have a screen width limit. If the screen width becomes goes bigger or smaller than the limit, then the grid layout will change.

These grids are a four based grid, each having the following class names:

  • .grid_1, which has a width 25% of its container
  • .grid_2, which has a width 50% of its container
  • .grid_3, which has a width 75% of its container
  • .grid_4, which has a width 100% of its container

Note: On this template, if the screen becomes smaller than 768px, all grids will have a 100% width

Below is an example of these grids:

Defining those grids requires you to declare a <div></div> tag, and give it the correct class name.

Note: It's recommended to always put the grids into the main container of the template.

Example
<div class="grid_2">
    <div class="da-panel collapsible">
        <div class="da-panel-header">
            <span class="da-panel-title">
                <img src="your-icon-url" />
                Panel Label
            </span>
        </div>
        <div class="da-panel-content">
        </div>
    </div>
</div>

The above code snippet will render a panel, like the second row on the preview image above.

Panel Content

Panel content are panels that has a header, it looks like the content blocks on the preview image above.

A panel has the following HTML structure:

<div class="da-panel collapsible">

        <span class="da-panel-title">

            Panel Label
        </span>

    </div>
</div>

 

  1. Panels are <div></div> tags with the class .da-panel
  2. It can be collapsible, this will be automatically done by giving it the class .collapsible
  3. Panel title will be put in the panel header, a <div></div> tag with the class .da-panel-header
  4. This header will contain the title, the icon and a toggle button for collapsible panels
  5. Panel contents should be put into a <div></div> tag with the class .da-panel-content
  6. Panel contents can have padding, this can be achieved by giving panel content the class .with-padding like on the code example above. Remove the class if you want a panel without paddings.

Scrollable Panel Content

The most common problem on mobile devices, and even desktops are content, usually tables that expands horizontally overflowing the container.

This template provides a solution by adding horizontal scrollbars below a content panel, by explicitly setting a class name on the panel if its content might overflow.

There was also an option to automate this for every overflowing content, but it's better by setting it explicitly, because it will make the code more efficient, and will prevent any undesired results.

In order to use this functionality, you have to do the following steps:

  1. Include tinyscrollbar plugin
  2. Give the panel whose content might overflow an additional class, .scrollable
  3. MAKE SURE the template's core script is included (forgetting to include this script will actually make this template lose most of its functionality)
Example:
<script type="text/javascript" src="js/jquery-1.7.2.min.js"></script>
<script type="text/javascript" src="js/jquery.tinyscrollbar.min.js"></script>
<script type="text/javascript" src="js/core/dandelion.core.js"></script>

<div class="grid_2">
    <div class="da-panel scrollable">
        <div class="da-panel-header">
            <span class="da-panel-title">
                <img src="images/icons/color/calendar_2.png" alt="" />
                Panel Title
            </span>
        </div>
        <div class="da-panel-content"> <-- Put your content here -->
        </div>
    </div>
</div>

This will make sure that a panel is scrollable if its content overflows the container.

Note: If the content changes by DOM operations, make sure to update the scrollbars by calling:

.tinyscrollbar_update();

Any problem or bugs with these scrollable functionality, please contact me.

Custom Plugins

There are three custom plugins included on this template, they are all special builds for this template because there are no suitable plugins available to get the desired results :)

Circular Statistic Widget

This plugin will render, with the help of HTML5 Canvas an animated circular statistic widget.

This widget can help you display data as a percentage or as fraction, so it requires a value and a maximum value.

Internet Explorer users require to include excanvas javascript plugin to simulate the canvas element,

To ease the use this plugin, jquery-metadata plugin should be included. This was choosen over HTML5 data attribute because this template is build upon XHTML, so the page will be invalid when using HTML5 data attributes.

On the dashboard on the demo page, we have the following structure to render this widget:

<ul class="da-circular-stat-wrap">
    <li class="da-circular-stat {fillColor: '#a6d037', value: 55, maxValue: 98, label: 'Seeds Collected'}"></li>
</ul>

As you probably notice, this will render a statistic widget like the first one on the preview image above.

By using jquery-metadata plugin, options for a single widget can be specified on the class attribute, between the curly brackets, separated by commas, preventing the need to initialize the plugin on every element.

Using this plugin requires you to include the following files:

Form Styling

Form Layouts

  • Inline form layout, which has labels on the left side of the input
  • Block form layout, which has labels on top of the input
  • Column form layout, which has form inputs divided in columns

Form HTML Structure:
<form class="da-form">
    <div class="da-form-block">
        <div class="da-form-row">
            <label></label>
            <div class="da-form-item">
                <textarea></textarea>
            </div>
        </div>
    </div>
    <div class="da-button-row">
    </div>
</form>

The markup above, will result in a block form layout. To get inline form layout, change

<div class="da-form-block"></div>

to

<div class="da-form-inline"></div>

By not wrapping the form rows with either .da-form-block  or .da-form-inline, the form will be a block form, which is the default layout.

Columned Forms

This is a form layout that divides a row into eight equal sized grids. These grids can be combined, thus creating columns with a width of 1/8, 2/8, 3/8, 4/8, 5/8, 6/8, 7/8, and 8/8 of its container.

Using this layout can be done by appending one of the following classes into a form row:

  • .da-form-col-1-8
  • .da-form-col-2-8
  • .da-form-col-3-8
  • .da-form-col-4-8
  • .da-form-col-5-8
  • .da-form-col-6-8
  • .da-form-col-7-8
  • .da-form-col-8-8

Notes:

For the first column in a row, you should put an additional class name, .alpha that indicates that this is the first column in a row, so removing the left margin.

For the last column in a row, you should put an additional class name, .omega that indicates that this is the last column in a row, so removing the right margin.

<div class="da-form-row">
    <div class="da-form-col-3-8 alpha">
        <label>Form Col 3 - 8</label>
        <input type="text">
    </div>
    <div class="da-form-col-3-8">
        <label>Form Col 3 - 8</label>
        <input type="text">
    </div>
    <div class="da-form-col-2-8 omega">
        <label>Form Col 2 - 8</label>
        <input type="text">
    </div>
</div>

Fieldsets

You can also wrap the form in a <fieldset></fieldset> instead of a <div></div>

<form class="da-form">
    <fieldset class="da-form-block">
        <legend>Form Legend</legend>
        <div class="da-form-row">
            <label></label>
            <div class="da-form-item">
                <textarea></textarea>
            </div>
        </div>
    </fieldset>
    <div class="da-button-row">
    </div>
</form>

Mixing Block and Inline Form

You can have a form with mixed layouts by just giving a form row the class .da-form-block  or .da-form-inline.

<form class="da-form">
    <div class="da-form-row da-form-block"></div>
    <div class="da-form-row da-form-inline"></div>
</form>

Form Input Size

Form inputs can have three sizes, small, default and large. default is the default size for form inputs.

You can control the input size by giving the class attribute .small, .default , or .large  to the form item wrapper.

<div class="da-form-item large">
    <input type="text" />
</div>

For default sized form elements, you don't need to specify a class name.

Form List

You can create a vertical or horizontal list of form elements like radio buttons, checkboxes and even text fields using unordered list with the class name .da-form-list

<form class="da-form">
    <div class="da-form-block">
        <div class="da-form-row">
            <label></label>
            <div class="da-form-item">
                <ul class="da-form-list">
                    <li><input type="radio" name="gender" /> <label>Male</label></li>
                    <li><input type="radio" name="gender" /> <label>Female</label></li>
                </ul>
            </div>
        </div>
    </div>
    <div class="da-button-row">
    </div>
</form>

An inline, horizontal list can be created by giving an additional class name, .inline  to the unordered list.

<ul class="da-form-list inline"></ul>

Form Input Styling

Styling the form inputs doesn't require you to add any class attribute, every input element inside a .da-form tag will get styled.

Form Notes

Form notes are short descriptions that can be put above any form element.

To define a form note, we should declare a  <div></div>, a<span></span>, or any plain element with the class name .formNote above the form input:

Example:
<form class="da-form">
    <div class="da-form-block">
        <div class="da-form-row">
            <label>Text Input</label>
            <div class="da-form-item">
                <span class="formNote">This is a form note</span>
                <input type="text" />
            </div>
        </div>
    </div>
</form>

Inline Error Message

Form error message can also be displayed inline below the form element.

To define an inline error message, we should declare a  <div></div>, a<span></span>, or any plain element with the class .errorMessage below the form input:

<form class="da-form">
    <div class="da-form-block">
        <div class="da-form-row">
            <label>Text Input</label>
            <div class="da-form-item">
                <input type="text" />
                <span class="errorMessage">This is an error message</span>
            </div>
        </div>
    </div>
</form>

Form Message

Form messages can be an info message, an error message, a warning message and a success message.

The base css class for all messages is .da-message, just add .success, .error , .info , or .warning  to get different form messages.

HTML structure:
<div class="da-message error"></div>
<div class="da-message success"></div>

Sytyling file inputs works directly from the template, but you still needs you to include the necessary javascript file.

File input styles are already integrated into the core css stylesheets, so no need to include any additional css files. Styling can be done by specifying the class .da-custom-file to a file input element, and it will automatically get styled.

Example:

<script type="text/javascript" src="js/jquery-1.7.2.min.js"></script>
<script type="text/javascript" src="js/jquery.fileinput.js"></script>

<input type="file" class="da-custom-file" />

The example above assumes you already included this template's core script.

Button Styling

Buttons are styled with CSS3 to be interactive and good looking.

  • There are 6 button colors
    • Red
    • Green
    • Blue
    • Pink
    • Orange
    • Gray
  • You can always add new colors by editing /css/core/buttons.css
  • To get the button style, just add the css class .da-button, to any clickable element you want like <a></a>, <button></button>, <input />
  • There are three different button sizes: large, medium (default) and small
  • Icons in buttons can only be done on <a></a> and <button></button> tags, to put icons on <input /> tags, you should define your own css class which sets an icon as its background-image
Example:
<button class="da-button red small">Click Me!</button>
<input type="submit" class="da-button blue" value="Submit" />

<script type="text/javascript" src="js/jquery-1.7.2.min.js"></script>
<script type="text/javascript" src="plugins/syntaxhighlighter/prettify/prettify.min.js"></script>
<link rel="stylesheet" href="plugins/syntaxhighlighter/prettify/prettify.min.css" media="screen" />

<script type="text/javascript" src="plugins/syntaxhighlighter/jquery.syntaxhighlighter.min.js"></script>
<link rel="stylesheet" href="plugins/syntaxhighlighter/styles/style.min.css" media="screen" />
<link rel="stylesheet" href="plugins/syntaxhighlighter/styles/theme-balupton.css" media="screen" />
<link rel="stylesheet" href="plugins/syntaxhighlighter/styles/theme-google.css" media="screen" />

<script type="text/javascript">
(function($) {
    $(document).ready(function(e) {
        if($.SyntaxHighlighter) {
            $.SyntaxHighlighter.init({
                alternateLines: true
            });
        }
    });
})(jQuery);
</script>

To highlight our code and auto-detect the language, we would simply add the classname .highlight to our code block, and it will automatically get styled.

<pre claas="highlight">
var self = this;
</pre>

More info about this plugin can be found here: https://github.com/balupton/jquery-syntaxhighlighter

Note: Live examples for this plugin is this documentation :)

Tipsy, a Tooltip Plugin

Tipsy is a jQuery plugin for creating a Facebook-like tooltips effect based on an anchor tag's title attribute.

This template integrates tooltips internally, so you only need to add the necessary javascript and css files and define the right css class for each element.

<script type="text/javascript" src="js/jquery-1.7.2.min.js"></script>
<script type="text/javascript" src="plugins/tipsy/jquery.tipsy.js"></script>
<link rel="stylesheet" type="text/css" href="plugins/tipsy/tipsy.css" media="screen" />

This template has eight build in classes, just add one on your element, and that element will display the tooltip when hovered.

.da-tooltip-nw .da-tooltip-n .da-tooltip-ne
.da-tooltip-w   .da-tooltip-e
.da-tooltip-sw .da-tooltip-s .da-tooltip-se

For more info about this plugin, please visit http://onehackoranother.com/projects/jquery/tipsy/

FAQ

How do I make a dropdown menu that’s always closed by default on the main navigation on the left?

Easy, just add the class “closed” to the second level ul tag of the menu.

Example:
<ul class="closed">
    <li><a href="grids.html">Grids and Panels</a></li>
    <li><a href="typography.html">Typography</a></li>
</ul>

How do I change the template's layout mode between fluid and fixed?

Please refer to sub-section Page Title, Main Wrapper and Logo from Building The Base HTML section for more detailed explanation.

Please look for a <div></div> tag with the id #da-wrapper, it might be located on top directly or not far after the opening <body></body> tag. This is the point where we need to decide whether our template will have a fixed or fluid width by default. Give this #da-wrapper div a class name: .fixed for fixed layout, or .fluid for fluid layout mode.

<body>
    <!--
        Main Wrapper. 
        Set this to 'fixed' for fixed layout and 'fluid' for fluid layout'
    -->
    <div id="da-wrapper" class="fluid">
    </div>
</body>
